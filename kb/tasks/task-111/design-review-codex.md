## Design Review: task-111 — 2026-07-11
- Reviewer: Codex
- Review status: request-changes
- Feedback:
  - Blocking: SNS preview/top-target API contract is underspecified. The design says `BuildPreview(state, def)` returns `TopTargetCustomerIds` (`kb/tasks/task-111/design.md:343`-`kb/tasks/task-111/design.md:345`), while F2 requires those labels to come from C4's top-2 customer archetypes and `ServiceManager.CustomerDefs` (`kb/tasks/task-111/design.md:425`-`kb/tasks/task-111/design.md:428`). But neither `SNSCampaignOps.BuildPreview` nor `ServiceManager.TryGetSnsPreview` is specified as passing the customer definitions into the pure calculation (`kb/tasks/task-111/design.md:343`, `kb/tasks/task-111/design.md:358`). This can lead to UI-side recomputation or an unimplementable DTO field, which would violate the "UI가 SO 배수를 직접 계산하지 않는다" dependency rule from task-110 (`kb/tasks/task-110/design.md:503`-`kb/tasks/task-110/design.md:505`). Fix by adding customer inputs to the preview/top-target pure helper, or by moving `TopTargetCustomerIds` construction explicitly into `ServiceManager` via a pure helper that receives projected `CustomerDefInput`.
  - Scope and hardcap alignment are otherwise sound. The design confines task-111 to one SNS campaign per Night and next-day effects only (`kb/tasks/task-111/design.md:38`-`kb/tasks/task-111/design.md:45`), keeps exactly three SNS channels/cost reseeds (`kb/tasks/task-111/design.md:121`-`kb/tasks/task-111/design.md:132`), excludes task-112 events and task-113 save/load implementation (`kb/tasks/task-111/design.md:60`), avoids new scenes/singleton managers (`kb/tasks/task-111/design.md:63`-`kb/tasks/task-111/design.md:65`), and preserves the two-scene hardcap (`kb/tasks/task-111/design.md:75`-`kb/tasks/task-111/design.md:76`). This matches demo scope's SNS row and hardcaps (`kb/concepts/demo-scope.md:22`, `kb/concepts/demo-scope.md:31`-`kb/concepts/demo-scope.md:35`).
  - Determinism contract is strong and task-110-compatible. The design reuses task-110 FNV-1a seed format unchanged (`kb/tasks/task-111/design.md:85`-`kb/tasks/task-111/design.md:88`; `kb/tasks/task-110/design.md:301`), keeps `RoundHalfUp`/milli integer projection (`kb/tasks/task-111/design.md:94`-`kb/tasks/task-111/design.md:96`, `kb/tasks/task-111/design.md:176`-`kb/tasks/task-111/design.md:189`; `kb/tasks/task-110/design.md:273`), requires explicit failure instead of neutral fallback (`kb/tasks/task-111/design.md:89`-`kb/tasks/task-111/design.md:91`), and defines base-prefix invariance (`kb/tasks/task-111/design.md:283`-`kb/tasks/task-111/design.md:286`). The C5 FNV vectors also preserve task-110's known vector and add useful SNS vectors (`kb/tasks/task-111/design.md:235`-`kb/tasks/task-111/design.md:241`; `kb/tasks/task-110/design.md:674`).
  - Public API/backward compatibility is adequately reflected. Existing `TryBuildDemandPlan` stays as a neutral overload (`kb/tasks/task-111/design.md:267`-`kb/tasks/task-111/design.md:282`), neutral orders remain unchanged (`kb/tasks/task-111/design.md:299`), and tests explicitly cover the old 4-input overload, task-110 deterministic baseline, and base-prefix invariance (`kb/tasks/task-111/design.md:634`-`kb/tasks/task-111/design.md:638`). The test gate also records and preserves the current EditMode 152/PlayMode 2 baseline (`kb/tasks/task-111/design.md:4`, `kb/tasks/task-111/design.md:608`-`kb/tasks/task-111/design.md:609`, `kb/tasks/task-111/design.md:674`-`kb/tasks/task-111/design.md:675`).
  - Test coverage is implementation-ready once the preview API gap is fixed. It covers record atomicity, malformed definitions/state, duplicate-night execution, stored-record consistency, modifier failures, JsonUtility round-trip, manager catalog failures, PlayMode day transition, balance guard, SceneBuilder idempotence, UI state, and smoke gates (`kb/tasks/task-111/design.md:618`-`kb/tasks/task-111/design.md:682`). The save/load system itself remains excluded; the JsonUtility round-trip is a schema determinism guard for task-113, not a save/load feature (`kb/tasks/task-111/design.md:648`-`kb/tasks/task-111/design.md:649`).
- Action required:
  - Update the SNS preview/top-target contract so the implementation has exactly one non-UI calculation path for `TopTargetCustomerIds`: either `BuildPreview(state, def, customers)`/equivalent projected inputs, or an explicit `ServiceManager` pure-helper path that receives `CustomerDefInput` and returns the preview DTO.
